Structure visuelle obligatoire
  • Aucune navigation métier (RM-21) — même coque que /login et /register.
  • Icône ou illustration de courrier envoyé.
  • Message clair : « Vérifiez votre boîte mail. Cliquez sur le lien de confirmation pour activer votre compte. »
  • Rappel de l'email saisi (les 3 premiers chars + domaine masqué partiellement : ex. cam***@gmail.com).
  • Bouton secondaire : « Renvoyer l'email » (avec anti-spam : désactivé 60 s après clic).
  • Lien retour vers /login.

Ce qui se passe côté email
  • Email envoyé avec un lien tokenisé horodaté.
  • Clic sur le lien → validation token → compte passe à ACTIVE → redirect /login.
  • Token expiré (> 24h) : message d'erreur avec bouton « Renvoyer un email » → variante reset-password-invalid-token.html comme référence UX.
États à prévoir
  • Default : message de confirmation avec rappel email.
  • Renvoi en cours : spinner sur le bouton renvoi.
  • Renvoi réussi : message « Email renvoyé. Vérifiez votre boîte mail. »
  • Anti-spam : bouton renvoi désactivé + countdown visible (60 s).
